# Рекомендации по производительности

## Как обеспечить быструю загрузку на мобильных

- Генерировать страницы статически.
- Держать клиентский JS около нуля на коммерческих страницах.
- Использовать только необходимые интерактивные компоненты.
- Загружать критичный контент первого экрана без ожидания JS.
- Не встраивать тяжелую карту, видео или карусели в hero.

## Изображения

- Использовать `AVIF` и `WebP` там, где это возможно.
- Задавать размеры изображений заранее.
- Делать responsive variants.
- Не хранить крупные hero-баннеры в PNG.
- Кейсы до/после оптимизировать отдельно под мобильные.

## Шрифты

- Self-hosted шрифты.
- 1-2 начертания на старте.
- Preload только реально критичных файлов.
- Не подключать внешние font CDN.

## JS и CSS

- Theme toggle реализовать легким скриптом.
- Сократить hydration до минимума.
- Использовать CSS variables вместо тяжелых JS-решений.
- Critical CSS для первого экрана.
- Минимизировать сторонние скрипты и виджеты.

## Как минимизировать риск медленного FCP и LCP

- Сделать hero как HTML + optimized image, без сложной анимации.
- Не блокировать первый экран шрифтами и тяжелым JS.
- Ленивая загрузка для карты, галерей и второстепенных изображений.
- Приоритетная загрузка только для LCP-изображения.
- Проверять мобильные метрики до релиза и после релиза.

## Допущение

- Основной трафик будет мобильным или смешанным, поэтому мобильная производительность важнее desktop-эффектов.
