# Редизайн главной и цветовой темы — «Фактура / 78»

Дата: 2026-04-12  
Статус: согласовано на уровне направления (ожидается ревью перед планом внедрения).

## Цель

Сделать главную страницу и визуальную систему ближе к практикам ниши **ремонт и химчистка обуви** (референсы: Sole Fresh СПб, Sneaknfresh, Sneakerklin), сохранив спокойную подачу без агрессивных модалок при входе.

## Принятые решения

### Направление контента и UX (гибрид, «вариант 3»)

1. **Герой:** материальность (фото мастерской или крупный план обуви) + **один сильный пример «до/после» в одном кадре** (сплит), в духе Sole Fresh.
2. **Кейсы:** ниже по скроллу — **1–3 интерактива с ползунком «до/после»** (как Sneakerklin), остальные работы — статичные карточки.
3. **Процесс:** секция **этапов чистки** (и при необходимости кратко — ремонта), нумерация и короткие подписи, **без набора заглавными буквами** и без перегруза; логика как у Sneaknfresh, подача спокойнее.
4. **Услуги:** блок с карточками/ценой «от» + ссылка на калькулятор и полный список услуг.
5. **Локация:** секция **карта + контакты + режим** в одном блоке.
6. **Запись и лиды:** CTA в потоке страницы (мессенджеры, телефон, при необходимости форма), **не модальное окно при первой загрузке**.

### Ограничения по промо (согласовано: B + C)

- **Запрещено:** полноэкранные и блокирующие модалки при входе; повторяющиеся агрессивные popup.
- **Допустимо:** узкая полоска под шапкой, компактный баннер между секциями, ненавязчивая плашка (например снизу), **с явным закрытием** и без перекрытия всего интерфейса; промо можно дублировать коротким текстом в hero или рядом с ценой.

### Цветовая гамма (ориентир Sneakerklin)

По автоматическому разбору встречающихся в разметке значений [sneakerklin.pro](https://sneakerklin.pro/) (ориентиры, не юридически «копировать пиксель в пиксель»):

| Роль (смысл)        | Примерные значения с референса | Назначение в нашей системе                          |
|---------------------|---------------------------------|-----------------------------------------------------|
| Фон тёмный          | `#222327`, `#111111`            | `--color-bg-base`, глубина героя и нижних секций   |
| Поверхности         | `#000` / уточнить оттенок       | `--color-bg-surface`, `--color-bg-elevated`        |
| Акцент «холодный»   | `#0077ff`, `#1d98dc`            | ссылки, вторичные кнопки, выделения                |
| Акцент «позитив»    | `#27D061`                       | успех, опционально вторичный CTA (осторожно с брендом) |
| Промо / внимание    | `#FFEB21`, `#FFE239`            | только крупные элементы, бейджи, полоски; мелкий текст на жёлтом — избегать без проверки контраста |
| Текст вторичный     | `#717275`, `#8b8b8b`            | `--color-text-secondary`, `--color-text-tertiary`  |
| Светлые панели      | `#f6f6f6`, `#ffffff`            | светлая тема или карточки на тёмном фоне по смыслу  |

**Текущее состояние проекта:** в `src/styles/global.css` задана **тёплая** крафт-палитра (беж/бронза). Редизайн предполагает **сдвиг к холодной тёмной базе и синим акцентам** в духе Sneakerklin, с **точечным** жёлтым/лаймовым для промо и важных меток, не заменяя читаемость и узнаваемость бренда «Фактура / 78» полным клонированием чужого бренда.

**Светлая тема:** остаётся переключаемой (`data-theme="light"`). Токены светлой темы нужно **пересобрать** так, чтобы она не конфликтовала с новой «холодной» акцентной парой (синий вместо текущей бронзы как основного интерактивного цвета), либо упростить светлую тему до нейтральной с той же акцентной логикой — решение зафиксировать при внедрении после визуального макета в браузере.

## Структура главной (порядок секций)

1. Header (существующий паттерн; при промо — место для узкой полоски под ним, опционально).
2. Hero: визуал + H1 про ремонт и химчистку обуви в СПб + краткий подзаголовок + CTA (фото / калькулятор).
3. «До/после» одним кадром (сплит).
4. Услуги (сетка карточек, усилить визуальную связь с обувью — иконки/фото по возможности).
5. «Как мы работаем» / этапы чистки (и кратко ремонт при необходимости).
6. Кейсы: ползунок на выбранных + сетка остальных + ссылка на все кейсы.
7. Доверие (существующий TrustBlock / отзывы — по текущему контенту).
8. FAQ.
9. Карта + контакты + CTA.
10. Footer.

Внутренние страницы в первую очередь **не ломаем**; после обновления токенов проверить контраст и читаемость на услугах и калькуляторе.

## Компоненты к реализации (новые или доработка)

| Компонент              | Описание                                                                 |
|------------------------|--------------------------------------------------------------------------|
| `Hero`                 | Слот под фото; сплит до/после; CTA                                       |
| `BeforeAfterSplit`     | Один кадр, два изображения, общая подпись                                |
| `BeforeAfterSlider`    | Ползунок сравнения, `prefers-reduced-motion` — статичный сплит           |
| `CleaningSteps`        | Сетка или горизонтальный скролл карточек шагов                           |
| `MapSection`           | Встраиваемая карта + адрес + часы + мессенджеры                          |
| `PromoStrip`           | Опционально, dismissible, без блокировки контента                      |

Зависимости: по возможности без тяжёлых библиотек; ползунок — нативный диапазон + pointer/keyboard.

## Нефункциональные требования

- **Доступность:** контраст текста к фону WCAG AA для основного текста; жёлтый — только крупные элементы или с тёмной подложкой.
- **Производительность:** ленивая загрузка изображений в hero и кейсах; не подключать тяжёлый виджет карты до секции в viewport (по возможности).
- **SEO и семантика:** один H1; секции с `section` и заголовками уровня 2–3.

## Вне скоупа первой итерации

- Модалки со скидкой при входе и полноэкранные перехваты курьера.
- Полный ребрендинг логотипа и шрифтов (при необходимости — отдельная задача).
